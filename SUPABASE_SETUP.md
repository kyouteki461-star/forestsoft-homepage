# Supabase 设置指南

## 问题诊断

您遇到的 CORS 错误表明需要在 Supabase 项目中配置正确的 CORS 设置。

## 解决步骤

### 1. 配置 CORS 设置

1. 打开 [Supabase 仪表板](https://supabase.com/dashboard/project/fzqbzujqjvmlowbjlxof)
2. 进入您的项目
3. 在左侧菜单中点击 **Settings** → **API**
4. 找到 **CORS Configuration** 部分
5. 添加以下域名到 Allowed origins 列表：
   ```
   http://localhost:3000
   http://127.0.0.1:3000
   http://localhost:8000
   https://kyouteki461-star.github.io
   ```
6. 点击 **Save**

### 2. 使用测试页面诊断问题

打开 `test-supabase.html` 页面，按顺序点击：
1. **测试连接** - 确认 Supabase 基本连接
2. **检查表是否存在** - 如果表不存在，需要创建
3. **插入示例数据** - 如果表存在但为空
4. **获取数据** - 验证数据是否正确

### 3. 创建 news 表

如果测试显示表不存在，请在 Supabase SQL Editor 中执行：

```sql
-- Create the news table
CREATE TABLE public.news (
    id INT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    title TEXT NOT NULL,
    date TEXT NOT NULL,
    content TEXT NOT NULL,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- Enable Row Level Security (RLS)
ALTER TABLE public.news ENABLE ROW LEVEL SECURITY;

-- Create a policy to allow public read access
CREATE POLICY "Public access to news" ON public.news
    FOR SELECT USING (true);

-- Create function to automatically update updated_at timestamp
CREATE OR REPLACE FUNCTION public.handle_updated_at()
RETURNS TRIGGER AS $$
BEGIN
    NEW.updated_at = NOW();
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

-- Create trigger to automatically update updated_at on row update
CREATE TRIGGER handle_news_updated_at
    BEFORE UPDATE ON public.news
    FOR EACH ROW
    EXECUTE FUNCTION public.handle_updated_at();
```

### 4. 使用修复后的代码

创建好表并配置好 CORS 后，使用修复后的文件：

1. 将 `js/supabase-config-fixed.js` 重命名为 `js/supabase-config.js`
2. 或者更新 news.html 中的引用

### 5. 常见问题

**Q: 仍然显示 CORS 错误？**
A: 确保已正确保存 CORS 设置，并清除浏览器缓存（Ctrl+Shift+R）

**Q: 表创建后仍然无法访问？**
A: 检查 RLS 策略是否正确创建，确保允许 SELECT 操作

**Q: 数据插入失败？**
A: 检查字段名和数据类型是否匹配

### 6. 验证安装

完成所有步骤后：
1. 打开 news.html
2. 如果显示新闻内容，说明成功
3. 双击页面进入编辑模式测试
