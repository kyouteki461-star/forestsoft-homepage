# Supabase + Vercel ニュース管理システム

このプロジェクトは、Vercel Serverless FunctionsとSupabaseを使ったニュース管理システムです。

## 構成

```
/
├── api/                 # API Routes (Serverless Functions)
│   ├── index.js         # メインAPIエンドポイント
│   ├── init-db.js       # データベース初期化API
│   └── news.js         # ニュース専用API
│   └── package.json     # Node.js依存関係
├── public/             # 静的ファイル
│   ├── index.html       # 古いUIテストページ
│   └── index-new.html  # 新しいUIページ
├── vercel.json         # Vercel設定
└── README.md           # このファイル
```

## APIエンドポイント

### 1. メインAPI (`/api/`)
- **GET** `/api/test-connection` - 接続テスト
- **GET** `/api/news` - ニュース一覧取得
- **POST** `/api/news` - ニュース作成

### 2. データベース初期化 (`/api/init-db`)
- **GET** `/api/init-db` - newsテーブルの作成

### 3. ニュース専用API (`/api/news`)
- **GET** `/api/news` - ニュース一覧取得
- **POST** `/api/news` - ニュース作成

## 環境変数

Vercelにデプロイする際は、以下の環境変数を設定してください：

```
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_SERVICE_ROLE_KEY=your-service-role-key
```

## デプロイ方法

### Vercel CLIでデプロイ

1. Vercel CLIをインストール
   ```bash
   npm i -g vercel
   ```

2. ログイン
   ```bash
   vercel login
   ```

3. デプロイ
   ```bash
   vercel
   ```

### GitHubとの連携

1. このリポジトリをGitHubにプッシュ
2. Vercelダッシュボードでリポジトリを連携
3. デプロイをトリガー

## 使い方

1. デプロイしたURLにアクセス
2. 「接続テスト」ボタンをクリックしてAPI接続を確認
3. 「テーブル確認」でデータベーステーブルの存在を確認
4. 「サンプルデータ挿入」でテストデータを作成
5. 「データ取得」でニュース一覧を表示

## Supabaseの設定

### テーブル作成（SQL）
```sql
-- newsテーブルの作成
CREATE TABLE public.news (
  id INT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  title TEXT NOT NULL,
  date TEXT NOT NULL,
  content TEXT NOT NULL,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- RLSの有効化
ALTER TABLE public.news ENABLE ROW LEVEL SECURITY;

-- 公開ポリシーの作成
CREATE POLICY "Public access to news" ON public.news
  FOR SELECT USING (true);
```

### APIキーの取得

1. Supabaseプロジェクトダッシュボードにアクセス
2. Settings > APIでURLとanon keyを確認
3. Settings > API > Project API Keysでservice_role keyを生成

## 注意点

- クライアントサイドから直接Supabaseクライアントを使用せず、必ずAPI経由でデータを取得/作成する
- CORSヘッダーが設定されているため、フロントエンドからのアクセスが許可されている
- VercelのEdge Functionsとの互換性のため、Node.js 20.xを使用
